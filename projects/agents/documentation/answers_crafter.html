Answers Crafter route

Overview

This API endpoint, /answers-crafter, takes a user query, validates it, conducts research, and writes content based on the research findings. The process is handled by an orchestrator that manages a pipeline of agents:

  • Validator: Validates and corrects the input.
  • Researcher: Conducts research on the latest AI trends.
  • Writer: Crafts engaging content based on the research findings.

API Endpoint

POST /answers-crafter

This endpoint processes the user query through a sequence of tasks performed by the AI agents.

Request Body: 
{
    "user_query": "What are the latest advancements in AI for 2024?"
}
Response: 
{
    "ai_agent_prompt": "Crafted article content based on research.",
    "pipeline_details": {
        "validated_input": "Input query validation and corrections.",
        "research_summary": "Summary of the latest AI research trends."
    }
}

Response Codes:

  • 200 OK: Successfully processed the query and returned the results.
  • 400 Bad Request: Missing 'user_query' in the request body.
  • 500 Internal Server Error: Something went wrong with the pipeline processing.

AI Agents

1. Validator

Role: Validate and correct the input for clarity and relevance.

System Message: "You are an AI specializing in response validation and correction. When given an input, assess the accuracy and relevance. If unclear or incorrect, provide a revised response that aligns with the input's requirements."

2. Researcher

Role: Conduct AI research to find the latest advancements and trends.

System Message: "You are an expert researcher in AI advancements. Given validated input, identify and summarize the latest trends and breakthroughs in AI, especially those from 2024."

3. Writer

Role: Craft engaging content based on research findings.

System Message: "You are a skilled writer who transforms research insights into engaging content. Take the summarized findings and create a compelling article suitable for a tech-savvy audience."

Task Workflow

The tasks are processed sequentially in the following order:

  1. Validate input: Ensures the user query is clear and relevant.
  2. Conduct research: Researches the latest advancements in AI, focusing on 2024 trends.
  3. Write content: Transforms the research into engaging, well-written content.

Error Handling

In case of errors, the API will return the following response structure:

Example Error Response: 
{
    "error": "User query is required"
}

The following errors can occur:

  • 400 Bad Request: Missing required parameters (e.g., user_query).
  • 500 Internal Server Error: Issues with processing the pipeline tasks.
